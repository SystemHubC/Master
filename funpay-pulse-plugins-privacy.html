Privacy Policy for FunPay Pulse Plugins

Last updated: 26 June 2026.

This policy applies to FunPay Pulse plugins published by SystemHubC / @AndreyCatser unless a plugin has its own separate privacy policy.

What data may be processed

Depending on plugin settings and enabled features, a plugin may process order identifiers, order status, buyer messages, seller messages, lot identifiers, lot prices, configured templates, API tokens saved by the seller, and technical logs needed for troubleshooting.

External services

Some plugins may contact third-party APIs configured by the seller, for example delivery providers, account marketplaces, notification services or other automation endpoints. Only the data required for the enabled action is sent to the configured service.

Storage

Plugin settings and operational state may be stored in the FunPay Pulse plugin storage assigned to the seller. Secrets such as API tokens are expected to be stored through Pulse secret storage when supported by the plugin.

Purpose

Data is used only to provide the selected automation: message replies, delivery actions, refunds, reviews, lot updates, notifications, diagnostics and related plugin functions.

Retention

The plugin does not intentionally sell personal data. Logs and state are kept only as long as needed for plugin operation, debugging, or until the seller removes the plugin data.
