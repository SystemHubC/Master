Политика конфиденциальности GPT Consultant GUI

Дата обновления: 26 июня 2026 года

Кратко: GPT Consultant GUI — плагин для FunPay Pulse, который может автоматически отвечать покупателям. Если владелец плагина включает LLM-режим, часть данных из чата и настроек магазина отправляется во внешний OpenAI-compatible endpoint, указанный владельцем плагина в настройках.

1. Какие данные обрабатывает плагин

Плагин может обрабатывать следующие данные из событий FunPay Pulse:

  • текст входящего сообщения покупателя;
  • идентификаторы события, сообщения, чата и аккаунта, необходимые для ответа и защиты от дублей;
  • имя пользователя покупателя, если оно передано событием FunPay Pulse;
  • контекст текущего лота: ID, название, описание и цена, если эти данные доступны в событии;
  • настройки владельца плагина: база знаний, системный промпт, включение/выключение LLM, модель, base URL LLM endpoint и лимиты ответа;
  • короткая локальная история диалога и заметки, если владелец плагина включил запоминание диалога или самообучение.

2. Передача данных во внешний LLM endpoint

Если настройка use_llm выключена, плагин использует локальный fallback и не отправляет текст чата во внешний LLM.

Если владелец плагина включает use_llm, плагин отправляет POST-запрос на OpenAI-compatible endpoint:

  • по умолчанию: https://api.openai.com/v1/chat/completions;
  • или на другой llm_base_url, который владелец плагина сам укажет в настройках.

В запрос может входить: системный промпт, база знаний продавца, контекст лота, последние сообщения диалога, если включено запоминание, и текущее сообщение покупателя. Это нужно только для генерации ответа покупателю.

3. Ключи доступа и секреты

Ключ доступа к LLM хранится через механизм секретов FunPay Pulse (secrets:own). Плагин не публикует ключ в сообщениях, не записывает его в README и не должен выводить его в публичные логи.

4. Хранение данных

Плагин может хранить собственное состояние через storage:own: обработанные события, время последнего ответа, короткую историю диалога и заметки самообучения. Эти данные нужны для работы функций антидубля, cooldown, памяти диалога и улучшения ответов.

Плагин не заявляет, что управляет политикой хранения данных у внешнего LLM-провайдера. Если владелец плагина указывает сторонний LLM endpoint, хранение и обработка данных на стороне этого endpoint регулируются политикой выбранного провайдера.

5. Цель обработки

Данные используются для автоматической консультации покупателей, формирования ответов по базе знаний продавца, контексту лота и истории текущего диалога, а также для технической диагностики работы плагина.

6. Что можно отключить

Владелец плагина может отключить LLM-режим, локальное запоминание диалога, самообучение и отправку ответов через настройки плагина FunPay Pulse.
